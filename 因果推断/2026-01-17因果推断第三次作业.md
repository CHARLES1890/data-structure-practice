### 1. 除广告和视频平台外，请介绍两个双边市场，及其供给方和需求方

在互联网业务中，双边市场的核心特征是平台作为中介，连接两组不同的用户，且一方的决策会影响另一方的结果。

**案例一：网约车平台（如滴滴、Uber）**

- **平台角色**：撮合出行需求与运力服务。
- **供给方：**司机（及车辆）**。他们提供运输服务，消耗的是时间和燃油 / 电费。
- **需求方**：**乘客**。他们支付费用，获取从 A 点到 B 点的位移服务。

**案例二：在线招聘平台（如 Boss 直聘、智联招聘）**

- **平台角色**：撮合人才与企业的用工需求。
- **供给方**：**求职者（简历 / 人才）**。他们提供劳动技能和时间。
- **需求方**：**企业（HR / 招聘方）**。他们提供薪资岗位，获取劳动力。

### 2. 简要阐述上述两个市场无法使用单边实验进行评估的原因

**核心痛点：外部性（Externality）与样本污染。**

如果只对需求方（如乘客）开启实验，而不对供给方（如司机）做任何处理，或者反之，会导致实验结果失真，无法归因。

**以网约车为例：**

假设我们要测试 “降低乘客端的打车价格” 对订单量的影响，仅在乘客端做 A/B 实验（部分乘客看到低价）：

1. **需求侧溢出**：实验组乘客看到低价，需求量激增。
2. **供给侧争抢**：激增的需求会去争抢池子里**有限的、未分组的司机**。
3. **对照组受损**：未享受优惠的对照组乘客会发现更难打到车（等待时间变长，被实验组抢走了司机），导致对照组体验下降，数据表现变差。
4. **结论偏差**：你会观察到实验组订单暴涨，但这不仅仅是因为价格降低，还因为它 “掠夺” 了对照组的资源。这种收益是以牺牲对照组为代价的，无法代表全量上线后的真实收益。

**以招聘平台为例：**

假设我们要测试 “给企业增加简历推荐量”，仅在企业端做实验：

1. **供给侧枯竭**：实验组企业收到了更多简历推荐，导致优质简历被这批企业迅速联系或锁定。
2. **需求侧反噬**：对照组企业（未被增加推荐）发现市场上的简历响应率变低、优质人才变少。
3. **结论偏差**：实验结果会虚高，因为它没有考虑到 “人才池” 总量是有限的。

**总结**：单边实验破坏了市场的自然平衡，导致**组间干扰（Interference）**，违反了随机对照试验（RCT）中 “独立同分布” 和 “互不干扰” 的基本假设。

### 3. 简要阐述对于上述两个市场如何使用双边实验进行评估

采用对偶实验设计或双层实验设计。

**核心策略：地理围栏法或双层正交实验。**

#### 方法一：基于 “市场” 的聚类随机化—— 适用于网约车、外卖等 LBS 业务

这是最常用的方法，核心是将 “供需双方绑定在同一个实验单元中”。

1. **定义实验单元**：不以 “人” 为单位，而是以 **“地理区域”**（如城市、行政区、甚至网格 Grid）为单位。
2. **分组逻辑**：
    
    - 将城市随机分为实验组城市和对照组城市。
    - **实验组城市**：该城市内的**所有乘客**看到新策略，该城市内的**所有司机**也执行新策略。
    - **对照组城市**：保持原有策略不变。
    
3. **评估逻辑**：
    
    - 比较实验组城市的整体 GMV、完单率、司机时薪。
    - 比较对照组城市的对应指标。
    
4. **优点**：完美隔离了供需影响，实验组的司机只服务实验组的乘客，不会产生跨组干扰。

#### 方法二：双层正交实验—— 适用于招聘、电商等非强 LBS 业务

当业务无法按地理切割时（例如招聘是全国性的），我们需要在用户 ID 层面做正交分层。

1. **分层逻辑**：
    
    - **Layer 1（需求层）**：对企业 ID 进行 Hash 取模，分为 A/B 组。
    - **Layer 2（供给层）**：对求职者 ID 进行 Hash 取模，分为 A/B 组。
    
2. **交叉组合（Cross-Product）**：
    
    - 最终形成四个象限：(企业 A, 求职者 A)、(企业 A, 求职者 B)、(企业 B, 求职者 A)、(企业 B, 求职者 B)。
    
3. **策略投放**：
    
    - 假设我们要测试 “新的匹配算法”。
    - 只有当**企业是实验组** 且 **求职者是实验组** 时，才触发新算法。
    - 其他组合（如企业 A 遇到求职者 B）走老逻辑（作为兜底）。
    
4. **评估逻辑**：
    
    - 分析 (企业 A, 求职者 A) 组合的连接效率（如回复率、入职率）。
    - 分析 (企业 B, 求职者 B) 组合作为基准。
    
5. **优点**：在不割裂整体大市场的情况下，实现了供需双边的同时控制，保证了统计显著性。

### 4.大盘当前有1000个实验，若这些策略实际上均没有效果，那在一类错误是成功控制在0.05的情况下，大盘显著实验有50个，此时大盘FDR是多少
 1. **一类错误（Type I Error, $\alpha$）**：原假设（$H_0$）为真（即策略无效）时，错误地拒绝了原假设（即误判为显著）的概率。题目中给出的控制目标是 0.05。
  * **假阳性（False Positives, FP）**：实际无效，但被检测为显著的实验数量。 
  * **真阳性（True Positives, TP）**：实际有效，且被检测为显著的实验数量。
  * **FDR 公式**： $$FDR = \frac{\text{假阳性数量 (FP)}}{\text{所有显著的实验数量 (FP + TP)}}$$
  2. 场景代入计算：
  -  **实际情况**：这 1000 个策略**实际上均没有效果**。 这意味着：**真阳性（TP）的数量为 0**。 
  -  **实验结果**：大盘显著的实验有 50 个。 这意味着：**所有显著的实验（50个）全部都是假阳性（FP）**。
  3.  最终计算 将数值代入 FDR 公式： $$FDR = \frac{50 \text{ (假阳性)}}{50 \text{ (假阳性)} + 0 \text{ (真阳性)}} = \frac{50}{50} = 1$$ 
  4. 结论与业务洞察 **结论**：此时大盘的 **FDR 为 100%**。
